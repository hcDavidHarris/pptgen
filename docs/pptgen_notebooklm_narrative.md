# pptgen -- Platform Narrative (NotebookLM Source)

## The Problem

Across engineering, analytics, and leadership teams, an enormous amount
of time is spent creating recurring presentation artifacts. Weekly
scorecards, quarterly reviews, architecture updates, DevOps metrics, and
leadership reports are often rebuilt manually in PowerPoint each time
they are needed.

The content changes, but the structure rarely does. Teams repeatedly
spend effort formatting slides, aligning layouts, and copying prior
decks instead of focusing on the information itself. This manual
workflow leads to inconsistency, duplicated effort, and slower
communication of important insights.

pptgen exists to solve this problem by separating **content from
presentation layout**.

## The Core Idea

pptgen is a platform that converts **structured content into
presentation‑ready PowerPoint decks**.

Instead of manually building slides, authors describe the presentation
in a simple YAML document. The YAML defines:

-   the slides
-   the content on each slide
-   the type of slide being rendered

A rendering engine then combines that structured content with an
approved PowerPoint template. The template defines the visual layout,
while the YAML defines the information. The result is a consistent,
repeatable slide deck generated automatically.

In short:

content → YAML → pptgen → PowerPoint

## Why This Matters

Separating content from formatting creates several important advantages.

First, presentations become **deterministic and reproducible**. The same
input always produces the same slide structure.

Second, organizations can enforce **design standards**. Templates
control branding, layout, and formatting so that every generated deck
remains consistent.

Third, presentations become **automatable artifacts**. Because YAML is
machine-readable, other systems---including AI tools---can generate the
content automatically.

## Real‑World Use Cases

The pptgen platform includes example libraries that demonstrate how
structured decks can support common organizational workflows.

Leadership teams can generate **EOS scorecards and quarterly reviews**.

Engineering organizations can generate **DevOps metrics dashboards and
transformation updates**.

Platform teams can communicate **Team Topologies and platform
architecture models**.

Architecture groups can convert **Architecture Decision Records (ADR)**
into presentation-ready review decks.

Each of these domains benefits from having a consistent slide structure
that is generated automatically from structured inputs.

## AI‑Assisted Deck Creation

One of the most powerful aspects of pptgen is that YAML decks can be
generated by AI.

Instead of manually writing the YAML file, a user can describe the
desired presentation in natural language. AI tools such as ChatGPT,
Claude, or Microsoft Copilot can translate that request into a valid
YAML deck definition.

pptgen then renders the final PowerPoint presentation from that
structured input.

This enables a workflow where:

idea → AI → YAML → pptgen → PowerPoint

In this model, humans focus on ideas and decisions, while the platform
handles formatting and presentation assembly.

## The Vision

pptgen is designed to become a reusable platform for generating
presentation artifacts across an organization.

By combining structured content, controlled templates, and AI-assisted
authoring, pptgen transforms presentations from manually assembled
documents into **repeatable, automated communication assets**.

Instead of rebuilding the same slides again and again, teams can define
their presentation patterns once---and let the platform generate the
rest.
